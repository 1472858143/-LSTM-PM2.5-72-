# 基于注意力机制 LSTM 的北京市 PM2.5 浓度 72 小时多步预测研究项目书

## 一、研究目标

本项目面向北京市 PM2.5 浓度未来 72 小时多步预测任务，构建统一的数据处理、模型训练、评价分析和结果输出系统，并通过传统时间序列模型、机器学习模型和深度学习模型的系统对比，重点验证 Attention-LSTM 在中长期预测、趋势拟合和污染峰值响应中的优势。

本项目核心目标包括：

1. 构建统一、可复现的小时级 PM2.5 预测实验系统。
2. 比较 `ARIMA`、`SARIMA`、`XGBoost`、`Random Forest`、`LSTM`、`Attention-LSTM` 六类模型。
3. 固定预测目标为未来连续 `72` 小时 `pm2_5`，统一采用直接多输出预测。
4. 新增不同输入窗口长度对比实验，分析更长历史序列对模型效果的影响。
5. 为毕业论文提供整体误差、分阶段误差、趋势拟合、峰值预测、长期稳定性和注意力解释的完整证据链。

## 二、任务定义与固定约束

### 2.1 输入特征

输入特征固定为 6 个，不允许增加或删减：

1. `temperature_2m`
2. `humidity`
3. `wind_speed_10m`
4. `precipitation`
5. `surface_pressure`
6. `pm2_5`

### 2.2 预测目标

输出目标固定为未来连续 `72` 小时 `pm2_5` 浓度：

$$
Y_t = [y_{t+1}, y_{t+2}, \ldots, y_{t+72}]
$$

### 2.3 多步预测方式

所有模型统一采用直接多输出预测，不允许改为递归预测。

### 2.4 数据划分

数据必须按时间顺序划分：

- 训练集：70%
- 验证集：15%
- 测试集：15%

禁止 shuffle，禁止测试集参与训练或调参。

### 2.5 固定评价指标

评价指标固定为：

- `RMSE`
- `MAE`
- `MAPE`
- `R2`
- `SMAPE`
- `MSE`
- `Explained Variance`
- `Max Error`

## 三、滑动窗口实验设计

在预测目标固定为未来 72 小时不变的前提下，本项目新增三组输入窗口长度对比实验：

| 实验名称 | 输入窗口 | 输出窗口 | 含义 |
| --- | --- | --- | --- |
| `input_168h` | 168h | 72h | 过去 7 天预测未来 72 小时 |
| `input_720h` | 720h | 72h | 过去 30 天预测未来 72 小时 |
| `input_2160h` | 2160h | 72h | 过去 90 天预测未来 72 小时 |

### 3.1 设计目的

新增窗口实验的目的不是改变任务定义，而是在统一预测目标下比较历史输入长度的影响：

1. `168h` 作为主实验基线。
2. `720h` 用于验证更长历史是否能提升中长期趋势学习。
3. `2160h` 用于验证超长历史是否能进一步提升 Attention-LSTM，或反而引入冗余信息。

### 3.2 论文分析重点

窗口长度实验需要重点回答以下问题：

1. 输入窗口长度对整体误差的影响。
2. 输入窗口长度对 `1-24h`、`25-48h`、`49-72h` 三阶段误差的影响。
3. Attention-LSTM 是否随输入窗口增长获得更明显收益。
4. 过长输入窗口是否带来冗余信息、训练成本上升和训练稳定性下降。

## 四、模型对比体系

本项目模型集合固定为：

1. `ARIMA`
2. `SARIMA`
3. `XGBoost`
4. `Random Forest`
5. `LSTM`
6. `Attention-LSTM`

### 4.1 ARIMA / SARIMA

- 仅使用 `pm2_5` 单变量序列
- 可忽略输入窗口对模型输入结构的影响
- 但输出目录仍按 `window_name` 分离，保持实验结构统一

### 4.2 XGBoost / Random Forest

输入采用滑动窗口展平形式，输入维度随窗口长度动态变化：

- `168h -> 168 × 6 = 1008`
- `720h -> 720 × 6 = 4320`
- `2160h -> 2160 × 6 = 12960`

代码中禁止硬编码 `1008`。

### 4.3 LSTM / Attention-LSTM

深度学习模型输入 shape 随窗口动态变化：

- `(B, 168, 6)`
- `(B, 720, 6)`
- `(B, 2160, 6)`

输出始终固定为：

$$
(B, 72)
$$

Attention-LSTM 的 `attention_weights` 长度必须与当前输入窗口一致，并在不同窗口实验中分别输出和可视化。

## 五、数据处理与工程约束

### 5.1 数据输入

输入数据为单一小时级 CSV，规范字段固定为：

- `timestamp`
- `pm2_5`
- `temperature_2m`
- `humidity`
- `wind_speed_10m`
- `precipitation`
- `surface_pressure`

### 5.2 处理原则

所有窗口实验共享同一套数据处理原则：

1. 字段映射与规范化。
2. 时间排序与重复时间处理。
3. 小时级重建索引。
4. 缺失值与异常值处理。
5. 仅用训练集拟合归一化参数。
6. 按时间顺序划分训练、验证、测试。

不同窗口实验之间必须重新构造滑动窗口样本，不允许共享窗口数组。

## 六、输出目录与文件结构

新版标准输出目录结构为：

```text
outputs/{window_name}/{model}/
```

例如：

```text
outputs/input_168h/attention_lstm/
outputs/input_720h/lstm/
outputs/input_2160h/xgboost/
```

每个模型目录中保留统一文件结构：

- `predictions.csv`
- `metrics.json`
- `stage_metrics.csv`
- `horizon_metrics.csv`
- `plots/`
- `model.pt`
- `config_snapshot.json`
- `training_log.json`

深度学习模型额外输出：

- `training_history.json`
- `training_history.csv`
- `plots/loss_curve.png`

Attention-LSTM 额外输出：

- `attention_weights.npy`
- `attention_stats.json`
- `plots/attention_weights.png`

其中 `predictions.csv` 列结构固定为：

```text
sample_id,timestamp,horizon,y_true,y_pred
```

不允许修改。

## 七、训练入口设计

训练入口统一为：

```powershell
python -m train.run_all --config config/config.json --models ... --windows ...
```

支持三种方式：

### 7.1 全部窗口、全部模型

```powershell
python -m train.run_all --config config/config.json --models all --windows all
```

### 7.2 指定单个窗口

```powershell
python -m train.run_all --config config/config.json --models all --windows input_168h
```

### 7.3 指定模型与窗口组合

```powershell
python -m train.run_all --config config/config.json --models lstm attention_lstm --windows input_720h input_2160h
```

若不传 `--windows`，默认运行 `window_experiments` 中 `enabled=true` 的全部窗口实验。

## 八、推荐实验流程

### 第一阶段：快速验证

```powershell
python -m train.run_all --config config/config.json --models random_forest xgboost --windows input_168h
```

### 第二阶段：核心深度模型对比

```powershell
python -m train.run_all --config config/config.json --models lstm attention_lstm --windows input_168h input_720h input_2160h
```

### 第三阶段：完整六模型实验

```powershell
python -m train.run_all --config config/config.json --models all --windows all
```

实验建议：

- `168h` 是主实验基线
- `720h` 和 `2160h` 只作为窗口长度影响验证
- 不建议默认加入 1 年或 2 年窗口，避免计算成本和冗余信息过高

## 九、评价与汇总分析

除单模型输出外，项目还必须生成跨窗口、跨模型的汇总结果：

```text
outputs/metrics_summary/window_model_metrics.csv
outputs/metrics_summary/stage_metrics_summary.csv
outputs/metrics_summary/horizon_metrics_summary.csv
```

主要用途：

1. 比较同一模型在不同输入窗口下的整体表现。
2. 比较同一模型在不同窗口下三阶段误差变化。
3. 比较同一模型误差随 horizon 增长的稳定性。
4. 判断 Attention-LSTM 是否在更长历史输入下取得更明显收益。

## 十、图表与解释要求

### 10.1 预测图时间轴

所有模型输出的预测曲线图必须使用真实 `timestamp` 作为 x 轴，不再使用 `time point`。

要求包括：

1. `prediction_curve.png` 的 x 轴来自 `predictions.csv` 中的 `timestamp`
2. `peak_case_top1.png` 同样使用真实时间轴
3. 自动格式化日期刻度并旋转标签，避免重叠
4. 图标题中包含：
   - `window_name`
   - `model_name`
   - `prediction target: 72h`

### 10.2 Attention 权重解释

Attention-LSTM 必须输出并分析不同窗口长度下的注意力权重，用于回答：

1. 更长窗口下注意力是否仍能聚焦关键时段
2. 注意力是否更关注最近时段、前一天同一时段或历史污染阶段
3. 窗口拉长后是否出现接近均匀分布的退化现象

## 十一、训练日志与可复现性

每个模型目录下的 `training_log.json` 必须记录：

1. `window_name`
2. `model_name`
3. `input_window_hours`
4. `output_window_hours`
5. `start_time`
6. `end_time`
7. `total_seconds`
8. `executed_steps`
9. `status`
10. `error_message`（失败时）

深度学习模型还需保留 epoch 级损失历史，用于分析收敛情况与 early stopping。

## 十二、结论支撑逻辑

本项目最终结论不能只依赖单一指标，而必须通过以下几个维度共同支撑：

1. 整体误差
2. 分阶段误差
3. 趋势拟合能力
4. 峰值预测能力
5. 长期稳定性
6. Attention 权重解释
7. 输入窗口长度对模型性能的影响

对于窗口长度实验，论文中应重点回答：

- `Attention-LSTM` 是否能从更长历史序列中提取有效信息
- 长窗口是否一定优于 168h 基线
- 过长窗口是否引入冗余并拖累训练效率与稳定性

## 十三、严格限制

以下核心约束不得修改：

1. 预测目标仍为未来 72 小时 `pm2_5`
2. 输入特征仍固定为 6 个
3. 模型集合仍固定为 6 类
4. 数据划分仍按时间顺序，禁止 shuffle
5. 评价指标仍固定为 8 项
6. `predictions.csv` 列结构不得改变
7. 必须保留旧版单窗口训练能力

## 总结

本项目书在原有 72 小时多步预测任务基础上，新增了 `168h`、`720h`、`2160h` 三组输入窗口长度对比实验，使研究不仅能比较模型类型，还能系统分析历史输入长度对不同模型，尤其是 `Attention-LSTM` 的影响。该设计能够更好支撑毕业论文关于长期依赖、趋势学习、污染峰值响应与模型稳定性的核心结论。
