# 01_项目开发总说明

## 1. 项目目标

本项目为“基于 Attention-LSTM 的北京市 PM2.5 浓度 72 小时多步预测系统”。项目目标是构建一个完整工程系统，覆盖数据处理、滑动窗口构造、模型训练、模型评估、结果可视化和前端展示，最终支撑毕业论文中的实验分析和研究结论。

系统需要完成以下目标：

1. 使用固定输入特征预测未来 72 小时 PM2.5 浓度。
2. 构建统一的数据处理与实验流程。
3. 实现传统时间序列模型、机器学习模型和深度学习模型的公平对比。
4. 重点验证 Attention-LSTM 在长时间预测、趋势拟合和污染峰值预测中的效果。
5. 输出统一格式的预测结果、评价指标和图表文件。
6. 构建正式前端展示模块，用于展示模型预测结果和实验对比结论。
7. 为毕业论文提供可复现实验过程、评价结果和分析依据。

## 2. 模型体系

本项目必须实现以下六类模型，不允许新增、删除或替换模型集合。

### 2.1 传统时间序列模型

1. ARIMA
2. SARIMA

ARIMA 和 SARIMA 作为传统统计模型基线，采用单变量 PM2.5 序列建模，用于判断仅依赖 PM2.5 历史序列时的预测能力。

### 2.2 机器学习模型

1. XGBoost
2. Random Forest

XGBoost 和 Random Forest 需要将滑动窗口样本转化为监督学习表格数据，即将输入窗口展平后进行多步预测。

### 2.3 深度学习模型

1. LSTM
2. Attention-LSTM

LSTM 和 Attention-LSTM 使用三维时间序列输入。Attention-LSTM 是核心模型，需要保存 Attention 权重，用于后续可视化与论文分析。

## 3. 已确定约束

以下内容已经由项目书明确确定，开发过程中必须严格执行，不允许修改、不允许替换、不允许重新设计。

### 3.1 输入特征

仅允许使用以下六个输入特征：

1. Temperature
2. Humidity
3. Wind Speed
4. Precipitation
5. Pressure
6. PM2.5

禁止新增风向、节假日、其他污染物、时间编码、站点编号等额外特征。

### 3.2 输出目标

输出目标固定为未来 72 小时 PM2.5 浓度。

输出向量格式为：

```text
(72,)
```

### 3.3 滑动窗口

滑动窗口固定如下：

1. 输入窗口：168 小时
2. 输出窗口：72 小时
3. 输入 shape：`(168, 6)`
4. 输出 shape：`(72,)`

### 3.4 多步预测方式

预测方式固定为直接多输出预测。

模型必须一次性输出未来 72 小时 PM2.5 预测值，禁止使用递归预测方式。

### 3.5 数据划分

数据集必须按时间顺序划分：

1. 训练集：70%
2. 验证集：15%
3. 测试集：15%

禁止 shuffle，禁止随机划分，禁止测试集参与归一化拟合、调参或训练。

### 3.6 评价指标

所有模型必须实现以下全部指标：

1. RMSE
2. MAE
3. MAPE，分母必须大于等于 1
4. R²
5. SMAPE
6. MSE
7. Explained Variance
8. Max Error

### 3.7 多步分析

所有模型必须进行以下多步分析：

1. 整体误差分析
2. 分阶段误差分析：1-24 小时、25-48 小时、49-72 小时
3. horizon 曲线分析：第 1 小时至第 72 小时逐步长误差变化

### 3.8 运行环境

运行环境固定要求如下：

1. GPU
2. PyTorch
3. `seed = 42`

### 3.9 前端系统

前端展示系统是正式系统模块，不是附加项。前端技术栈固定为：

1. Vue 3
2. ECharts

前端必须展示：

1. 模型选择
2. 预测曲线
3. 指标对比
4. 分阶段误差
5. Attention 可视化

## 6. 开发原则

### 6.1 项目书优先

凡是项目书中已经明确给出的内容，必须完全按照项目书执行。开发中不得为了方便实现而修改输入特征、窗口长度、模型集合、数据划分比例、预测方式或评价指标。

### 6.2 配置集中管理

窗口长度、数据路径、模型参数、训练参数、输出路径和随机种子应放入配置文件管理，禁止散落在多个脚本中。

### 6.3 模块职责清晰

数据处理、窗口构造、模型定义、训练、评估、可视化和前端展示必须分模块实现。不得将所有逻辑写入单一脚本。

### 6.4 输出格式统一

所有模型必须输出相同格式的预测结果和指标文件，便于统一评估、统一可视化和前端读取。

### 6.5 实验可复现

所有实验必须记录数据版本、模型参数、训练配置、随机种子、输出路径和评价指标，保证论文结果可以复现。

### 6.6 结论由实验支撑

不得只凭单个指标得出结论。Attention-LSTM 的有效性必须通过整体误差、分阶段误差、horizon 曲线、峰值预测、趋势拟合和 Attention 权重共同支撑。

## 7. 论文导向说明

本项目开发不是单纯完成代码运行，而是为毕业论文提供实验支撑。代码开发必须服务于以下论文分析目标：

1. 说明为什么使用 168 小时输入窗口和 72 小时输出窗口。
2. 说明传统时间序列模型、机器学习模型和深度学习模型的差异。
3. 证明直接多输出预测适合 72 小时 PM2.5 多步预测任务。
4. 对比 ARIMA、SARIMA、XGBoost、Random Forest、LSTM 和 Attention-LSTM 的预测效果。
5. 分析 Attention-LSTM 是否在长时间预测中更稳定。
6. 分析 Attention-LSTM 是否在趋势拟合和峰值预测中更优。
7. 使用 Attention 权重辅助解释模型关注的关键历史时间步。
8. 通过前端系统展示实验结果，形成完整工程闭环。
